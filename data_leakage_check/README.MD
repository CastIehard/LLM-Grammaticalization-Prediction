# DataвЂ‘Leakage Checks for LLMs

Tools to (1) build a likely **leaked subset** of your test data by scanning large corpora and (2) run **leakage-detection experiments** on local HF models (TinyLlama, FLANвЂ‘T5) and OpenAI models (gptвЂ‘4o, gptвЂ‘4oвЂ‘mini).

---

## What this repo does

- **Build leaked subsets** by streaming public corpora (Wikipedia, C4/mC4, The Pile) and finding overlaps with your evaluation keywords.
- **Probe models for leakage** with a suite of methods:
  - **M1 вЂ“ Partial Keyword Recall**
  - **ICL Recall вЂ“ InвЂ‘Context Learning variant of M1**
  - **M2 вЂ“ Real vs. Fake Keyword Completion**
  - **HalfвЂ‘Set Context Recall**
  - **M3 вЂ“ ScoreвЂ‘Based вЂњSuggest AnotherвЂќ**
  - **M4 вЂ“ Predict Score from Keyword**
- **Log metrics + plots** per model Г— dataset to the `results/` folder.

## Quick start

### 1) (Optional) Build a leaked subset
This scans a big source (Wikipedia/C4/mC4/Pile) and saves matching keywords with a snippet.

```bash
python build_leaked_subset.py
```

**Edit inside the script**:
- `SOURCE = "wikipedia" | "c4" | "pile"`
- For Wikipedia: `WIKI_CONFIG = "20231101.de"` (or `.en`)
- For C4/mC4: set `C4_LANG = "en"` or `"de"`
- Paths: `TESTSET_ORIGINAL`, `OUT_CSV`
- Limits: `MAX_ROWS_TO_SCAN`, `MAX_UNIQUE_KEYWORDS`

Output CSVs (examples already included):
- `leaked_from_source_c4.csv`
- `leaked_from_source_c4_de.csv`
- `leaked_from_source_pile.csv`

## Methods & metrics (why these help)

- **M1 вЂ“ Partial Recall**: Can the model restore a keyword from `[x___]`?  
  High recall hints at prior exposure to the exact tokens.
- **ICL Recall**: Same as M1 but after giving a small list of known examples.  
  If recall spikes here, the model may be patternвЂ‘matching memorized neighbors.
- **M2 вЂ“ Real vs Fake Gap**: Compare completions for true prepositions vs GermanвЂ‘ish nonвЂ‘words.  
  A large Realв€’Fake gap is a classic leakage signal.
- **HalfвЂ‘Set Context**: Give half the test set as context, test the other half.  
  Helps separate *onвЂ‘theвЂ‘fly priming* from *pretraining memory*.
- **M3 вЂ“ ScoreвЂ‘Matching вЂњSuggest AnotherвЂќ**: With a few examples for a score bucket, can the model name another true item?  
  Elevated match rate can indicate exposure to the labeled inventory.
- **M4 вЂ“ Predict Score from Keyword**: Ask for the grammaticalization level (1вЂ“4).  
  Alignment with ground truth can be due to generalization or label memorization; interpret together with other methods.

All metrics aggregate into `results/all_runs_summary.csv` and perвЂ‘run plots.

## Troubleshooting

- **File parsing issues**: The loader tries TSV/CSV with/without headers. If it still fails, open your file and ensure thereвЂ™s a column of strings (keywords) and a column containing digits 1вЂ“4.
- **OpenAI errors**: Ensure `openai` & `pythonвЂ‘dotenv` are installed, `.env` exists, and the key is active.
